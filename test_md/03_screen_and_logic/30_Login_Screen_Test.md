# Test 30. Login Screen Functional Test

## 1. 테스트 목적
- 로그인 프로세스 및 계정 상태에 따른 분기 처리 정상 작동 확인.

## 2. 테스트 시나리오
| ID | 테스트 항목 | 기대 결과 | 통과 여부 |
| :--- | :--- | :--- | :--- |
| LGN-01 | 정상 로그인 | 아이디와 비밀번호 일치 및 상태가 `일반회원` 또는 `관리자`일 때 메인 대시보드로 이동 | 통과 |
| LGN-02 | 잘못된 정보 입력 | 일치하지 않는 아이디/비밀번호 입력 시 에러 메시지 노출 | 통과 |
| LGN-03 | 대기 계정 처리 | 가입 신청 직후 `대기` 상태에서 로그인 시 `관리자의 승인을 기다려주세요.` 에러 노출 | 통과 |
| LGN-04 | 거절 계정 처리 | 승인 거절된 `거절` 상태에서 로그인 시 `가입이 거절된 계정입니다.` 에러 노출 | 통과 |
| LGN-05 | 이미 로그인된 상태 | 로그인 된 유저가 `/login` 접근 시 자동으로 메인 대시보드로 리다이렉트됨 | 통과 |

## 3. 검증 환경
- 테스트 계정: Dummy 계정 다수 (상태별)
- 에러 처리: Supabase `.single()` 통신 에러 완벽 캐치
- 모든 React 린트 오류 및 타입 에러 수정 완료 후 통과됨
