# Test 30. Login Screen Functional Test

## 1. 테스트 목적
- 로그인 프로세스의 보안성 및 편의 기능 정상 작동 확인.

## 2. 테스트 시나리오
| ID | 테스트 항목 | 기대 결과 | 통과 여부 |
| :--- | :--- | :--- | :--- |
| LGN-01 | 이메일 기억하기 | 체크 후 로그인 성공 시, 재접속할 때 이메일이 자동 입력됨 | 통과 |
| LGN-02 | Caps Lock 감지 | Caps Lock 활성화 시 실시간 경고 메시지가 노출됨 | 통과 |
| LGN-03 | 미인증 계정 처리 | 이메일 미인증 상태로 로그인 시 `이메일 인증이 필요합니다.` 에러 노출 | 통과 |
| LGN-04 | 승인 대기 계정 | 인증 완료 후 `status: PENDING` 상태에서 로그인 시 대기 안내 메시지 노출 | 대기 |
| LGN-05 | 감사 로그 확인 | 로그인 성공 직후 `audit_logs`에 IP와 브라우저 정보가 기록됨 | 통과 |

## 3. 검증 환경
- 브라우저: iOS Safari, Android Chrome
