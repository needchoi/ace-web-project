# Test 04. E2E Functional Test Plan (통합 E2E 테스트 계획서)

## 1. 테스트 목적
- ACE Football Manager v4.8의 회원가입부터 관리자 승인, 로그인, 대시보드 진입까지 이어지는 전체 워크플로우(End-to-End)를 검증합니다.
- 로컬 환경(`http://localhost:5173`) 및 배포 환경(`https://ace-footballclub.vercel.app`) 양측에서 동일한 기능성을 보장하는지 확인합니다.

## 2. 테스트 환경
- **대상 환경**: 
  - Local Dev Server
  - Vercel Production
- **검증 도구**: E2E 브라우저 자동화 테스트 (Subagent)

## 3. 통합 테스트 시나리오 (E2E-01)
| 단계 | 액션 (Action) | 기대 결과 (Expected Result) |
| :--- | :--- | :--- |
| **1. 회원가입 접근** | `/signup` 페이지로 이동 | 폼이 정상적으로 렌더링됨 |
| **2. 유효성 검사** | 아무 정보 입력 없이 '가입 신청하기' 클릭 | 필수값 누락 에러 메시지 노출됨 |
| **3. 정상 가입** | 신규 데이터 입력 후 제출 (`test_e2e`, `1234` 등) | "가입 신청 완료" 안내 컴포넌트 노출됨 |
| **4. 대기자 로그인** | `/login` 으로 이동 후 방금 생성한 계정으로 로그인 | "관리자의 승인을 기다려주세요." 에러 노출됨 |
| **5. 관리자 로그인** | `관리자` 권한을 가진 세션으로 강제 부여 또는 정상 로그인 후 `/admin` 이동 | 관리자 대시보드 및 가입 대기자 목록 렌더링됨 |
| **6. 가입 승인** | 리스트에서 방금 생성한 `test_e2e` 계정의 '승인하기' 버튼 클릭 | 대기 명단에서 유저가 즉시 사라짐 |
| **7. 승인 계정 로그인** | 기존 세션 로그아웃 후 `test_e2e` 계정으로 다시 `/login` | 로그인 성공 후 대시보드(`/`)로 자동 이동됨 |
| **8. 대시보드 확인** | 대시보드 진입 후 UI 상태 확인 | "환영합니다! {이름}님", Status: `일반회원` 정상 표기됨 |
