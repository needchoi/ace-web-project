# 33. Dashboard (대시보드 - 초기 Placeholder 버전)

## 1. UI 구성 요소 (Claude Sonnet Style)

### 1.1 메인 화면 (Coming Soon View)
- **전체 레이아웃**: 모바일 화면 중앙에 정갈하게 배치된 단일 카드 구조.
- **메시지 영역**:
  - **타이틀**: "COMING SOON" (정갈한 Serif 또는 Sans-serif 폰트, 자간 넓게).
  - **설명**: "서비스를 준비 중입니다. 관리자의 승인이 완료되었습니다. 잠시만 기다려 주세요."
- **브랜드 요소**: 화면 하단에 `ACE Football Manager` 로고를 아주 연한 그레이 톤으로 배치.

### 1.2 상단 헤더 (Global Header)
- **로고**: 좌측 상단에 위치.
- **테마 토글**: 라이트/다크 모드 전환 버튼.
- **로그아웃 버튼**: 우측 상단에 '로그아웃' 텍스트 또는 아이콘 배치.

## 2. 비즈니스 로직 (Business Logic)

### 2.1 진입 조건 (Entry Guard)
- **조건**: 사용자가 로그인 상태(`Authenticated`)이며, `status === 'ACTIVE'`인 경우에만 이 화면을 렌더링.
- **자동 리다이렉트**: 로그인하지 않은 사용자가 대시보드 접근 시 로그인 페이지(`#login`)로 강제 이동.

### 2.2 로그아웃 처리
- '로그아웃' 클릭 시 `supabase.auth.signOut()` 호출 및 로컬 세션 삭제 후 로그인 화면으로 이동.

## 3. 데이터 흐름 (Data Flow)
- **Fetch**: 로그인한 사용자의 프로필 정보(Name, Role)를 가져와 환영 문구에 활용.
  - 예: "환영합니다, {name}님! 현재 서비스를 준비 중입니다."
