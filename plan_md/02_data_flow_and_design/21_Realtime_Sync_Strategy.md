# 21. Realtime Sync & UI Strategy (실시간 동기화 및 UI 전략)

## 1. 실시간 동기화 메커니즘
본 시스템은 Supabase Realtime(PostgreSQL CDC)을 활용하여 다중 사용자의 데이터 일관성을 유지합니다.

## 2. 주요 실시간 시나리오
1. **전술 보드 (Tactical Board)**
   - 상황: 코치가 선수 배치를 변경하면 모든 스태프 화면에 즉시 반영.
   - 구현: `supabase.channel('tactics')`를 통해 실시간 브로드캐스팅.
   - 충돌 해결: Last-Write-Wins 전략을 기본으로 하되, 중요 변경 시 변경자 정보(Avatar)를 일시적으로 표시.

2. **경기 스코어 및 타임라인**
   - 상황: 경기 중 득점이나 교체 발생 시 대시보드 알림.
   - 구현: `matches` 테이블의 `UPDATE` 이벤트를 구독하여 토스트 메시지 출력.

## 3. 낙관적 업데이트 (Optimistic UI) 설계
사용자의 체감 속도를 높이기 위해 서버 응답 전 UI를 먼저 업데이트합니다.
- **적용 대상**: 능력치 수정, 포지션 드래그 앤 드롭, 공지사항 등록.
- **프로세스**: 
  1. UI 상태 업데이트 (로컬)
  2. 서버 API 호출
  3. 성공 시 유지 / 실패 시 이전 상태로 롤백 및 에러 알림.

## 4. 네트워크 단절 대응
- 네트워크 오프라인 상태 발생 시 상단에 '오프라인 모드' 바 노출.
- 변경된 데이터를 로컬 큐(Queue)에 쌓아두고 재연결 시 일괄 전송(Sync) 시도.
