# 05. API Design Specification (오픈 API 설계 원칙)

## 1. 개방형 요청 (Open Access)
- 시스템 내의 모든 API 요청은 인증(Auth) 정보나 세션 토큰 없이 이루어집니다.
- 누구나 읽기/쓰기가 가능한 상태를 전제로 API가 설계됩니다.

## 2. CRUD 인터페이스 (Supabase Auto-API)
- **Flexible Endpoints**: 데이터베이스에 테이블이 생성됨에 따라 Supabase가 자동으로 제공하는 REST API 엔드포인트를 그대로 사용합니다.
- **No User Filtering**: 사용자 식별자가 존재하지 않으므로, 데이터 조회 시 '내 데이터'를 필터링하는 조건(`user_id=eq...`)은 사용하지 않습니다. 
- 데이터를 필터링해야 하는 경우에는 데이터 자체의 속성(예: '진행 예정인 경기', '2026년 시즌 기록')만을 기준으로 필터링합니다.

## 3. 현재 사양
- 현재 데이터베이스는 초기화 상태이며, 기능 구현에 맞춰 공용 API 엔드포인트를 정의하고 호출할 예정입니다.
